# 이미지 업로드 기능 구현

## 1. 개요
게시판 게시글에 이미지 파일을 첨부가능, 이미지 업로드 기능의 추가.  
이미지 파일 자체는 Laravel Storage를 통해 서버 로컬 스토리지에 저장하고,  
DB에는 이미지의 경로 및 메타데이터를 저장하는 구조로 설계한다.


## 2. 이미지 모델 및 마이그레이션 생성

### 마이그레이션 생성
```bash
php artisan make:migration create_post_images_table
```


## 3. 스토리지 공개 링크
```bash
php artisan storage:link
```

pulbic\storage 에 심볼릭 링크 설정
- 실제 로컬 저장 위치는 storage/app/public에 저장  
- 브라우저에서 접근할 링크 설정하는것 (public\storage)


## 4. 게시글 작성 폼에 추가

multipart/form-data 설정

폼에서 파일 받기 위한 선언부
create.blade에 우선 선언, 나중에 edit.blade도 고치기

PostController에 /images 로 request 받는거 구현

## 5. PostController 처리 로직

1. 게시글 제목/내용 검증
2. 이미지 파일 유효성 검증
3. 게시글 저장
4. 업로드 파일 public\ 저장
5. 저장된 경로 및 파일데이터 post_images 테이블에 저장

***이때 DB::transaction() 파사드로 오류 발생시 롤백 가능한 처리 구현***

**Post - PostImage 개념상 1:N (hasmany - belongsTo), 다만 비즈니스 로직상은 한개만 매핑했음**


## 6. 게시글에 기존 속성들 이미지도 함께 매핑
