# AdminLTE UI 적용 가이드라인


## 1. 공통 레이아웃 분리 
- 기존 CRUD 페이지에서 공통 HTML 구조를 분리한다.
- `layouts/admin.blade.php`를 생성하고,
  navbar / sidebar / footer / alerts를 partial로 나눈다.
- 개별 페이지는 `@extends('layouts.admin')` 구조로 전환한다.

## 2. 공통 레이아웃에 AdminLTE 우선 적용
- AdminLTE starter/layout 예제를 기준으로 뼈대를 구성한다.
- CSS/JS는 우선 `public/adminlte` + `asset()` 방식으로 연결한다.
- content 영역은 `@yield('content')`로 비워두고,
  레이아웃만 정상 표시되는지 먼저 검증한다.

## 3. 로그인 페이지 별도 이식
- 로그인 화면은 `layouts/auth.blade.php`로 분리한다.
- AdminLTE login 예제를 Blade로 옮긴 뒤,
  불필요한 링크와 데모 문구를 제거한다.
- Laravel 로그인 route, `@csrf`, validation 메시지를 연결한다.

## 4. 게시글 목록 페이지 이식
- 목록 화면은 AdminLTE card + table 구조로 변환한다.
- 컨트롤러는 `paginate()`를 사용하고,
  뷰는 `links()`로 페이지네이션을 출력한다.
- 작성 버튼, 상태 메시지, 수정/삭제 버튼을 함께 배치한다.

## 5. 게시글 작성/수정/상세 페이지 이식
- form/card 중심의 UI로 변환한다.
- 입력 검증 에러 표시, 버튼 정렬, 업로드 입력을 정리한다.
- 목록 페이지와 동일한 공용 레이아웃을 사용한다.

## 6. 적용 원칙
- 기능 로직보다 UI 레이어만 먼저 바꾼다.
- HTML 예제는 그대로 쓰지 않고 Blade 문법으로 변환한다.
- 디자인 검증과 데이터 연결을 한 번에 하지 않는다.
- 로그인 화면과 관리자 화면의 레이아웃은 분리한다.
